Microchip Support
Library Interface
a) Initialization Function
 
Name 
Description 
 
TWIHSx_Initialize 
Initializes the instance of the FLEXCOM peripheral operating in TWI mode. 
b) Setup Functions
 
Name 
Description 
 
TWIHSx_CallbackRegister 
Sets the pointer to the function (and it's context) to be called when the given TWIHS's transfer events occur. 
c) Transaction Functions
 
Name 
Description 
 
TWIHSx_ReadByte 
Read the received TWI byte 
 
TWIHSx_WriteByte 
Write a data byte to TWI master 
 
TWIHSx_NACKDataPhase 
Configures the hardware to send NAK for the next data phase 
d) Status Functions
 
Name 
Description 
 
TWIHSx_IsBusy 
Returns the Peripheral busy status. 
 
TWIHSx_LastByteAckStatusGet 
Returns the ACK status of the last byte written to the TWI master. 
 
TWIHSx_StatusGet 
Returns the TWI hardware status flags. 
 
TWIHSx_TransferDirGet 
Returns the TWI transfer direction. 
e) Data Types and Constants
 
Name 
Description 
 
TWIHS_SLAVE_ACK_STATUS 
Defines the enum for the TWI slave acknowledgement. 
 
TWIHS_SLAVE_CALLBACK 
Defines the data type and function signature for the TWIHS Slave callback function. 
 
TWIHS_SLAVE_STATUS_FLAG 
This enum defines the list of possible TWI slave events 
 
TWIHS_SLAVE_TRANSFER_DIR 
Defines the enum for TWI data transfer direction. 
 
TWIHS_SLAVE_TRANSFER_EVENT 
Defines the enum for the TWI slave transfer event.
