SERCOMx_USART_Write Function

This function submits a write buffer to the SERCOM_USART peripheral to transfer. The data to be transmitted is deep copied into the SERCOM_USART peripheral's transmit ring buffer. The size of this buffer is configurable in MHC. Since, the data is deep copied into the transmit buffer, it allows the use of local variables/buffers. 

This function call is always non-blocking. Call to this function copies the application's transmit buffer into the internal transmit ring buffer and returns immediately. If notifications are enabled and a callback is registered, a callback is given when the threshold condition is met.

C
size_t SERCOMx_USART_Write(
    uint8_t* pWrBuffer, 
    const size_t size
);
Preconditions

SERCOMx_USART_Initialize must have been called for the associated SERCOM_USART instance.

Parameters
Parameters 
Description 
pWrBuffer 
Pointer to the user buffer. 
size 
Number of bytes to be transferred. 
Returns

The API returns the number of bytes actually copied. Depending on the space available in the transmit buffer, the number of bytes copied may be less than or equal to the requested size. Application must retry by calling this API again if the return value is less than the number of bytes requested. Application can also use the SERCOMx_USART_WriteFreeBufferCountGet API to determine the free space available in the transmit buffer before calling this API. Application can also choose to register for event notification to get notified when there is sufficient free space (using the SERCOMx_USART_WriteThresholdSet API) in the transmit buffer.

Remarks

This API is only available when SERCOM_USART ring buffer mode is enabled in MHC.

Example
size_t nBytesCopied;
uint8_t txBuffer[50];
uint32_t nBytes;

// Check if there is any free space is available in the transmit buffer
if (SERCOM1_USART_WriteFreeBufferCountGet() > 0)
{
    nBytesCopied = SERCOM1_USART_Write((uint8_t*)txBuffer, nBytes);  
    
    if (nBytesCopied != nBytes)
    {
        //The transmit buffer is full. Try after some time.  
    }
}
